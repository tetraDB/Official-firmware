Notifies teh face that the user is dragging their finder on the screen an passes in a bunch of info for the method to calculate exactly what the users intent is, which is quite complicated.

Clicks are now handled outside of the face (in display.cpp) so what we try to determine here is if the user intends to swipe teh face to a new one, and what options there are to do that fom this faces navigation options.

When a face is being dragged or swiped to a new face, as it's moving, the face is also responsible to moving the adjacent face.

My plan is to clean this all up/optimise/refine this - eventually
